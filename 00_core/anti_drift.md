# ANTI-DRIFTThis file exists to stop the DAMAGE repo from turning into generic aesthetic mush.## Primary drift threatsThe repo is at risk of drifting into:- generic glitchcore- generic lo-fi- generic retro nostalgia- "corrupted aesthetic" buzzword sludge- decorative dirt overlays- cyberpunk interface distortion- horror static as a catchall- fake VHS shorthand with no specificity## Common weak language to avoidThe following terms are too vague on their own:- glitchy- corrupted- retro- broken- damaged- noisy- lo-fi- analog- old-school- distortedThese words are allowed only if they are followed by specificity.Example:- weak: "glitchy VHS look"- stronger: "unstable VHS playback with chroma bleed, head-switching noise, and horizontal tracking tear"## Major confusion zones### 1. Glitchcore driftProblem: treating all digital breakage as one aesthetic bucket.Fix: Differentiate:- macroblocking- datamosh residue- packet loss- edge ringing- posterization- file corruption- scanline capture aliasing### 2. Retro fetish driftProblem: using old-media signifiers without actual artifact logic.Fix: Document specific behavior:- tape skew- phosphor bloom- gate weave- chroma crawl- copy-of-copy decay### 3. Dirt-overlay driftProblem: treating dust, grain, scratches, and fog as interchangeable dirt.Fix: Separate:- dust on sensor- dust in projection path- emulsion scratches- lens grease- scan dust- xerox toner voids### 4. Blur mush driftProblem: collapsing all softness into one blur bucket.Fix: Distinguish:- defocus- diffusion- smear- bloom- halation- motion drag- codec smearing- chroma bleed### 5. Horror shorthand driftProblem: assuming all damaged media implies dread.Fix: Allow many affective modes:- forensic- melancholic- ecstatic- nostalgic- psychedelic- clinical- sacred- filthy- sublimeDamage is not only horror.## Repo discipline rulesWhen writing a file, do not stop at:- what it feels like- what era it suggests- what style bucket it resemblesAlways also define:- the medium- the mechanism- the visible behavior- the confusion risks- common stack partners## Fast correction ruleIf a description could apply equally well to:- VHS- film- JPEG- CRT- dirty optics- print damagethen it is probably too vague and needs tightening.## Anti-drift testBefore finalizing a concept, ask:1. Could this belong to a specific medium?2. Could a generator or analyst use this description directly?3. Does it identify actual artifact behavior?4. Does it avoid vague trend-language?5. Does it meaningfully differ from neighboring concepts?If not, tighten it.
