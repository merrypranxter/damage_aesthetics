# ARTIFACT STACK LOGICThis file explains how different damage artifacts combine, reinforce, clash, or transform one another.Most real damaged media does not contain a single isolated effect.It contains **stacks**.A stack is a layered combination of damage behaviors that coexist within the same image, video, frame sequence, or playback system.## Why stack logic mattersWithout stack logic, damage design becomes shallow and incoherent.You get:- one-note glitch effects- fake-looking combinations- vague prompt mush- contradictory artifact pairings- visual noise without medium coherenceWith stack logic, you get:- believable failure histories- richer visual complexity- more controlled legibility- medium-specific realism- better prompt assembly- stronger shader / FX design## What makes a stackA stack may emerge from:- one medium with multiple natural artifacts- one cause producing many symptoms- a transfer chain with multiple failure points- one medium captured through another- deliberate hybrid simulation- repeated recompression / replay / duplicationExamples:- VHS tape + CRT display + phone camera capture + social-media recompression- scratched negative + dust + light leak + gate weave- dirty lens + overexposure + sensor clipping- JPEG compression + resave decay + screen recording## Core stack types### 1. Native stacksArtifacts that naturally co-occur within one medium.Example: VHS native stack- soft luma detail- chroma bleed- tracking instability- head-switching noise- horizontal jitterExample: film native stack- grain- dust- scratches- gate weave- halation- light leakThese stacks should feel internally coherent.### 2. Causal stacksArtifacts produced by one root cause.Example: dirty lens causal stack- haze veil- contrast loss- flare contamination- glow around highlights- smeared edge softnessExample: heat damage causal stack- warping- color shift- material buckling- exposure instability- partial melt or burn marksThe point here is symptom clustering.### 3. Chain stacksArtifacts accumulated through a sequence of media handoffs.Example:original digital video →played on CRT →filmed with phone camera →uploaded to platform →downloaded →screen-recorded againPossible stack:- CRT scanline interference- moiré- phone sensor noise- rolling shutter- compression artifacts- transcode banding- screen-capture softnessChain stacks are some of the richest and ugliest in the best way.### 4. Hybrid stylized stacksDeliberately combined artifacts from multiple systems for expressive effect.Example:- film grain- datamosh residue- phosphor bloom- optical grease haze- packet-loss fragmentationThese are not necessarily "real" in strict documentary terms, but they can still feel coherent if the families are chosen well.## Stack compatibilityNot all artifacts combine equally well.Some combinations reinforce each other.Some neutralize each other.Some confuse the medium identity.## Reinforcing combinationsThese tend to work well because they share visual or causal logic.### Optical contamination + exposure failure- greasy lens + bloom- haze + flare- condensation + veiling glare### Analog video + sync instability- tape smear + horizontal tear- chroma bleed + jitter- tracking error + noisy luma bands### Compression + temporal artifacts- macroblocks + interframe residue- packet loss + freeze loops- datamosh + motion echoes### Film wear + material contamination- scratches + dust- grain + gate weave- light leaks + chemical stain### Web decay + recompression- screenshot-of-screenshot softness- ringing + block breakup- color simplification + text edge chatter## Tension combinationsThese can work, but need careful handling.### Film grain + ultra-clean digital compressionPossible, but may feel incoherent unless framed as a hybrid chain.### Optical diffusion + hard-edged macroblockingCan be great, but needs intent. One softens; one hardens.### CRT curvature + phone rolling shutter + heavy datamoshPossible in a media-chain context, but can get visually overcrowded.### Dense scratches + aggressive block corruptionBoth are high-information damage systems; together they may cause clutter.## Contradictory combinationsSome combinations are not impossible, but become suspicious or muddy if not acknowledged.Examples:- perfect VHS tracking with ultra-clean CRT geometry and also highly modern interframe melt- realistic dirty lens behavior combined with artifact types that would only appear after several unrelated digital steps, with no chain logic- analog film burn paired with digital block corruption but no indication of scanning, compression, or transferThese can still be used artistically, but should be marked as hybrids, not presented as naturally native.## Stack dimensionsEvery stack should be considered across multiple dimensions:### 1. Medium coherenceDo the chosen artifacts plausibly belong together?### 2. Causal coherenceCould one process reasonably generate these effects?### 3. Temporal coherenceDo these effects happen at the same time scale?### 4. Severity balanceIs one effect dominating too much or flattening the others?### 5. Legibility loadHow much visual information is being obscured or scrambled?### 6. Surface densityIs the frame too cluttered with micro-artifacts?### 7. Signal identityDoes the image still feel like it belongs to a recognizable system?## Stack design heuristics### Rule 1: Start with a base mediumPick the anchor first:- VHS- film- JPEG- CRT- lens contamination- scan / xerox- streaming video- phone capture### Rule 2: Add native companionsChoose artifacts that normally belong with that medium.### Rule 3: Add one cause-based modifierSuch as:- dirty optics- heat- age- duplication- bad transfer- poor storage- replay instability### Rule 4: Add one optional chain contaminantSuch as:- recompression- screen capture- transcode failure- social-media upload damage### Rule 5: Check legibility before adding moreDo not keep piling shit on until the image becomes meaningless unless that is explicitly the goal.## Readability zones for stacks### Subtle stack2–3 low-level artifacts, mostly atmospheric.Example:- faint grain- light chroma softness- slight haze### Medium stack3–5 artifacts with one dominant family.Example:- VHS softness- chroma bleed- horizontal jitter- head-switching noise### Dense stack5+ interacting artifacts with clear system logic.Example:- CRT capture moiré- phone sensor noise- rolling shutter- compression breakup- edge ringing- light glare### Collapse stackSevere overlapping damage that partially destroys legibility.Example:- packet loss- freeze loops- macroblocks- datamosh residue- sync discontinuity- severe motion confusion## Stack writing templateWhen describing a stack, include:1. anchor medium2. secondary medium or chain if relevant3. primary artifact family4. secondary artifact families5. severity6. legibility impact7. visual mood8. realism vs stylization level## SummaryA good damage stack should feel like a **history of mishandling, transmission, or material struggle.**Not just:"throw every glitch effect in the kitchen sink and hope it looks cool."That path leads to ugly bullshit for no reason.
