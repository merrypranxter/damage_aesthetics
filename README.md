# DAMAGE**DAMAGE** is a cross-media taxonomy of visual degradation, artifact behavior, contamination, corruption, and failure signatures across analog, digital, optical, chemical, and playback systems.This is not merely a "style repo." It is a structured system for understanding, describing, organizing, and generating the aesthetics of broken media.DAMAGE includes:- analog video failures- digital compression artifacts- film wear and chemical damage- dirty optics and lens contamination- CRT and display defects- print / scan / xerox decay- sensor failures- playback instability- web recompression damage- temporal ghosting and memory scars- deliberate simulation of believable media failureThe core idea is simple:> Media does not fail randomly. It fails according to the properties of its format, material, capture chain, storage conditions, playback device, and transmission method.This repo treats those failures as a usable visual language.## What DAMAGE isDAMAGE is:- medium-specific- cause-aware- behavior-based- stackable- useful for prompting, shader design, visual analysis, and taxonomy buildingDAMAGE is concerned with:- what kind of artifact appears- why it appears- what medium it belongs to- how it behaves visually- what other artifacts it commonly coexists with- how strongly it affects legibility, mood, and perception## What DAMAGE is notDAMAGE is not:- generic glitchcore- random RGB split slapped on everything- "old media vibes" with no process truth- a one-click lo-fi filter pack- generic horror static- vague "corrupted aesthetic" language with no medium specificityThis repo avoids empty aesthetic mush by staying grounded in actual artifact families, real or believable causes, and cross-medium comparison.## Repo goalsThis repo exists to:1. define a complete cross-media taxonomy of damage behaviors2. separate artifacts by medium, family, and cause3. document artifact stacking logic4. provide structured prompting language5. support shader / FX / app logic6. preserve realism where desired7. support deliberate stylization without losing causal coherence## Major domainsThe repo is organized around:- **core theory**- **taxonomy**- **media formats**- **damage families**- **processes and causes**- **stacks and profiles**- **color and texture**- **composition**- **prompting**- **shader and FX translation**- **machine-readable logic**- **examples and reference intake**## Core principleA damaged image is not just "fucked up."It has a failure history.DAMAGE treats visible artifacts as signatures of:- storage- time- handling- optics- heat- signal weakness- compression- duplication- decoding failure- material contamination- playback instability- system mismatchThat history is the point.## Primary use casesThis repo can be used for:- AI image prompting- AI video prompting- shader / post-FX systems- style engines- media analysis- artifact simulation- creative coding- app taxonomy and tagging- forensic visual breakdown- hybrid analog/digital aesthetic design## Guiding philosophyDAMAGE is about **injured media.**Not clean nostalgia.Not generic distortion.Not decorative dirt.It is the structure of failure made visible.## Suggested reading order1. `00_core/damage_definition.md`2. `00_core/scope_and_boundaries.md`3. `01_taxonomy/master_taxonomy.md`4.
